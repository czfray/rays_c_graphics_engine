\doxysubsection{Detailed Description}
Shader management. 

This module allows the creating and using of shaders. Shaders are programs run in the GPU that determines how graphics are rendered on screen. They are necessary for all graphics rendering, even for simple unlit meshes.

To initialise a shader, you must:\+


\begin{DoxyEnumerate}
\item Create shader (See \doxylink{group__shaders_ga59204e5d6ef1ff3c6a6c533020fdf3ce}{rcge\+\_\+shader\+\_\+create})
\item Attach shader components (See section \doxysectlink{group__shaders_shaders_comp}{Shader Components and Pipeline}{2} on what components are)
\item Set outputs (See section \doxysectlink{group__shaders_shaders_colorout}{Shaders Output and Color Buffer}{2} on what outputs are)
\item Compile (See \doxylink{group__shaders_ga790ff83c14919e66b916c92931423426}{rcge\+\_\+shader\+\_\+compile})
\item Set attributes \& uniforms (See section \doxysectlink{group__shaders_shaders_attrib}{Shaders Attributes}{2} and \doxysectlink{group__shaders_shaders_uniform}{Shaders Uniforms}{2} on what attributes and uniforms are)
\item Use shader (See \doxylink{group__shaders_ga0303317abc7fcb74551d4777b665d987}{rcge\+\_\+shader\+\_\+use})
\end{DoxyEnumerate}

An example has been provided in section \doxysectlink{group__shaders_shaders_example}{Shader Setup Sample Code}{2}. \DoxyHorRuler{0}
 \hypertarget{group__shaders_shaders_comp}{}\doxysubsubsection{\texorpdfstring{Shader Components and Pipeline}{Shader Components and Pipeline}}\label{group__shaders_shaders_comp}
A shader is needed to be consisted of at least 2 components:\+ a vertex, and a fragment component. If you want, you can additionally make use of a geometric component.

Vertex shader component:\+ Processes each vertex to determine its position in 3D space and pass data to the next stage. Geometric Shader Component:\+ Takes whole primitives and can create, modify, or discard geometry before shape assembly. Fragment Shader Component:\+ Calculates the final color and appearance of each pixel on the screen.

When mesh vertices data (i.\+e. Raw data of a 2D shape/\+3D model) are passed to the GPU for rendering, it goes through the shader component by component. The pipeline is as follows:\+
\begin{DoxyEnumerate}
\item Passes data of vertices to vertex component of the shader
\item Passes output of vertex component to geometric component of the shader (optional)
\item Assembles shapes and rasterizes with the output data from the vertex component and the optional geometric component (Automatically done by the engine, no components needed)
\item Passes the shape data resulted to the fragment component of the shader
\item Renders the colors obtained from the shader to the screen
\end{DoxyEnumerate}

 \hypertarget{group__shaders_shaders_comp_source}{}\doxysubsubsection{\texorpdfstring{Shader Component Code (GLSL)}{Shader Component Code (GLSL)}}\label{group__shaders_shaders_comp_source}
In order to create your own shader component, you will need to write {\bfseries{GLSL (Open\+GL Shader Language)}} source code for it. Then, call \doxylink{group__shaders_ga5e647894c3ff7042acc70b3b62151da3}{rcge\+\_\+shader\+\_\+comp\+\_\+create} to load it into the engine.

A brief guide of GLSL {\itshape (Credit:\+ Learn Open\+GL, by Joey de Vries)}:\+~\newline
GLSL is tailored for use with graphics and contains useful features specifically targeted at vector and matrix manipulation. Shaders always begin with a version declaration, followed by a list of input and output variables, uniforms and its main function. Each shader\textquotesingle{}s entry point is at its main function where we process any input variables and output the results in its output variables. Don\textquotesingle{}t worry if you don\textquotesingle{}t know what uniforms are, we\textquotesingle{}ll get to those shortly.

\end{DoxyItemize}\hypertarget{group__shaders_shaders_attrib}{}\doxysubsubsection{\texorpdfstring{Shaders Attributes}{Shaders Attributes}}\label{group__shaders_shaders_attrib}
Attributes are the mesh vertices input into the shader. Since the inputs are first passed through the vertex component, attributes can also be defined as the inputs of the shader vertex component. So basically, attributes are to provide per-\/vertex data like position, normals, texture coordinates, and colors that define the shape and surface of a mesh. Attributes are usually used to input position, and texture UV coordinates data of vertices of a mesh.

Suppose I have a vertex shader component with source code as following:\+


\begin{DoxyCode}{0}

\DoxyCodeLine{}
\DoxyCodeLine{in\ vec3\ position;}
\DoxyCodeLine{in\ vec2\ texcoord;}
\DoxyCodeLine{...}
\DoxyCodeLine{}
\DoxyCodeLine{void\ main()}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ ...}
\DoxyCodeLine{\}}

\end{DoxyCode}
 Then the shader with this component would have two attributes:\+ "{}position"{} and "{}texcoord"{}.

 \hypertarget{group__shaders_shaders_uniform}{}\doxysubsubsection{\texorpdfstring{Shaders Uniforms}{Shaders Uniforms}}\label{group__shaders_shaders_uniform}
Uniforms are another type of input of a shader. It is a shader input that is constant for the entire draw call (the same value is used for all vertices and fragments). (Attributes, on the other hand, is an input that changes per vertex.) Uniforms are usually used for global data that affects the whole object being drawn instead of just single vertices, like transformation matrices (model/\+view/\+projection), lighting parameters, time values, and texture settings.

Suppose I have a shader component with source code as following:\+


\begin{DoxyCode}{0}

\DoxyCodeLine{}
\DoxyCodeLine{in\ vec3\ position;}
\DoxyCodeLine{...}
\DoxyCodeLine{uniform\ mat4\ model;}
\DoxyCodeLine{uniform\ mat4\ view;}
\DoxyCodeLine{uniform\ mat4\ proj;}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ main()}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ ...}
\DoxyCodeLine{\}}

\end{DoxyCode}
 Then the shader with this component would have uniforms "{}model"{}, "{}view"{} and "{}proj"{}, all of type mat4.
