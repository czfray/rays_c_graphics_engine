\doxysection{Elements and Element Managers }
\hypertarget{group__elements}{}\label{group__elements}\index{Elements and Element Managers@{Elements and Element Managers}}


\doxysubsection{Detailed Description}
Elements (RCGE object) management. 

\hypertarget{group__elements_elements_obj}{}\doxysubsubsection{\texorpdfstring{Elements}{Elements}}\label{group__elements_elements_obj}
An element groups multiple components (like meshes or sound sources) together, allowing a more organised way to manage objects that work together.

e.\+g. If 6 meshes makes up a player model, you can make a "{}player"{} element, attach 6 meshes to the element, and manage them together. Originally, you have to call \doxylink{group__meshes_gade5d248ba4125f12dfa6b3b8ad5c0616}{rcge\+\_\+mesh\+\_\+draw} for every element per frame. It makes the code very messy.

If you manage them under one element, you can group the draws in one update callback and simply call 1 function \doxylink{group__elements_ga66287a114895b04cfdac050e59910aad}{rcge\+\_\+element\+\_\+update} per frame, making the code way cleaner.

By adding component deletion calls in the deletion callback of an element, components can be automatically deleted when their parent element are deleted as well.\hypertarget{group__elements_elements_manager}{}\doxysubsubsection{\texorpdfstring{Element Manager}{Element Manager}}\label{group__elements_elements_manager}
If you have created a lot of elements, it becomes tedious to call \doxylink{group__elements_ga66287a114895b04cfdac050e59910aad}{rcge\+\_\+element\+\_\+update} for all element every frame. You do not want 100 lines of the same code right?

An element manager groups elements together, so that you do not have to update every element one by one per frame. Instead, after you create an element, you could simply attach the element to a manager, and simply call a single function \doxylink{group__elements_ga6ccf6875a32888495ad705984f58c47d}{rcge\+\_\+element\+\_\+manager\+\_\+update} per frame to update all elements. This also allows the automatic deletion of elements in the manager when the manager itself is deleted. \doxysubsubsection*{Typedefs}

Deletes an element. 

Element deletion must be done before the program terminates. Prevent memory leaks.

\begin{DoxyNote}{Note}
-\/ This function calls the element\textquotesingle{}s delete callbacks. Please delete the components in the callback, so if the element is deleted, it\textquotesingle{}s components are automatically deleted as well. 

-\/ If you created an object with a element manager, you do not have to call this. The element manager deletes the element for you when the program terminates. 
\end{DoxyNote}
